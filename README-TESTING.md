# Cooper Cookbook Testing Documentation

This project implements a comprehensive testing strategy covering both Backend (Java/Spring Boot) and Frontend (React/Vite).

## Backend Testing (JUnit 5 + Mockito + JaCoCo)

The backend test suite verifies core business logic, security filters, and database interactions using an in-memory H2 database.

### Test Categories

| Component | Tools | Focus |
|:---|:---|:---|
| **Controllers** | `MockMvc`, `@WebMvcTest` | API endpoints, status codes, JSON serialization. |
| **Services** | `Mockito`, `JUnit 5` | Business logic, forking rules, password verification. |
| **Repositories** | `JdbcTest`, `H2` | SQL queries, schema integrity, key generation. |
| **Security** | `JwtUtil`, `JwtFilter` | Token generation, parsing, and context authorization. |

### Key Test Classes
- `AuthControllerTest`: registration, login, and step-1 validation.
- `RecipeControllerTest`: CRUD operations and forking endpoints.
- `UserControllerTest`: Profile management and security verification.
- `CollectionRepositoryTest`: Integration testing with H2 database.
- `JwtUtilTest`: Cryptographic token validation.

### Limit Testing
We specifically test edge cases including:
- Duplicate usernames/emails.
- Private recipes in fork lineages (verified they display as `(Privated)`).
- Extremely long tags and special characters in the `TagInput`.
- Invalid password attempts in Security Management.

### How to Run Backend Tests & Coverage
```bash
# Run all tests
./mvnw test

# View coverage report (generated by JaCoCo)
# open target/site/jacoco/index.html
```

## Frontend Testing (Vitest + React Testing Library)

The frontend uses Vitest for high-performance unit and component testing.

### Component Coverage
- **`Navbar`**: Ensures the navigation is hidden for logged-out users.
- **`RecipeCard`**: Verifies dynamic metadata rendering (e.g., Forked vs. Original).
- **`TagInput`**: Comprehensive testing of keyboard events (Enter to add, Backspace to remove).

### Limit Testing (Frontend)
- **Tag Volume**: Verified handling of extremely long strings (1000+ chars) in tags.
- **Deduplication**: Logic verified to prevent duplicate tags in the UI state.
- **Redirects**: Verified immediate redirect to `/browse` upon login.

### How to Run Frontend Tests
```bash
cd cookbook-frontend
npm test
```

## Coverage Goal
The project maintains **100% logic coverage** on critical paths including Authentication, Forking Lineage, and Stash Management.
